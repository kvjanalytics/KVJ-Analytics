Introduction to Errors in Programming

In real-world programming, errors are unavoidable. Even experienced developers encounter errors while building software systems.

A good programmer must be able to:

  • Identify errors
  • Understand why they occur
  • Fix them efficiently
  • Prevent programs from crashing
Business Context:
• A financial system should not crash if a user enters invalid data
• A sales program should handle incorrect inputs safely
• A reporting system must detect problems before processing files

Python provides tools for debugging and handling errors gracefully.

5.1 Types of Errors

Python programs can encounter three main types of errors:

  1. Syntax Errors
  2. Runtime Errors
  3. Logic Errors

Syntax Errors

Syntax errors occur when Python code does not follow the language rules.

Example:

if 10 > 5
    print("Yes")

Output

SyntaxError: expected ':'
Explanation: The colon : is missing after the if condition. Python cannot interpret the statement and stops execution.

Runtime Errors

Runtime errors occur while the program is running.

Example:

print(10 / 0)

Output

ZeroDivisionError: division by zero
Explanation: Division by zero is mathematically impossible. The program crashes during execution.

Logic Errors

Logic errors occur when the program runs without crashing but produces incorrect results.

Example:

def add(a, b):
    return a - b

print(add(5, 3))

Output

2
The function is named add, but it performs subtraction. Python does not detect this error because the syntax is correct.

5.2 Exception Handling

Instead of allowing a program to crash, Python provides exception handling. This allows programs to handle errors safely using: try, except, else, and finally.

Basic try–except

Example:

try:
    print(10 / 0)
except:
    print("An error occurred")

Output

An error occurred
Explanation: The risky code is placed inside the try block. If an error occurs, Python executes the except block.

Handling Specific Exceptions

It is better to catch specific errors rather than all errors.

Example:

try:
    number = int("abc")
except ValueError:
    print("Invalid number conversion")

Output

Invalid number conversion
Explanation: int("abc") cannot convert text into an integer. Python raises a ValueError, which we handle using except ValueError.

try – except – else

The else block executes only if no error occurs.

Example:

try:
    result = 10 / 2
except ZeroDivisionError:
    print("Cannot divide by zero")
else:
    print("Result:", result)

Output

Result: 5.0

finally Block

The finally block always executes, whether an error occurs or not.

Example:

try:
    print(10 / 2)
except:
    print("Error occurred")
finally:
    print("Program execution finished")

Output

5.0
Program execution finished
The finally block is often used to release resources, such as closing files.

raise Keyword

The raise statement allows programmers to generate custom exceptions.

Example:

age = -5
if age < 0:
    raise ValueError("Age cannot be negative")

Output

ValueError: Age cannot be negative

5.3 Unit Testing

Unit testing verifies that individual parts of a program work correctly. Python provides the unittest module for automated testing.

Basic Unit Test Structure

Example:

import unittest

class TestMath(unittest.TestCase):
    def test_add(self):
        self.assertEqual(2 + 3, 5)

if __name__ == "__main__":
    unittest.main()
Explanation: assertEqual() checks whether the result matches the expected value. If correct, the test passes.

Common Unit Testing Assertions

  • assertTrue(condition): Passes if the condition is True.
  • assertIsInstance(val, type): Verifies if the variable belongs to the expected data type.
  • assertIn(val, list): Checks whether a value exists inside a collection.
  • assertEqual(a, b): Ensures that the expected output equals the actual output.

Integrated Business Example

Example: Safe financial division.

def divide_profit(total_profit, partners):
    if partners == 0:
        raise ValueError("Partners cannot be zero")
    return total_profit / partners

try:
    print(divide_profit(10000, 2))
except ValueError as error:
    print(error)

Output

5000.0
Explanation: This example demonstrates: Custom error validation, Exception handling, and Safe calculation. Such validation is important in financial applications.
